# 风格 D · 瑞士网格风「CHEN YU®」设计规范

> 原型：`style-d-swiss.html` · 默认浅色模式
> 关键词：网格外露、克莱因蓝、超大字、编号系统、零圆角

## 1. 设计原则

这一版继承国际主义平面风格（Swiss / International Typographic Style）：网格不是隐形的脚手架，而是可见的设计本体。页面由 1px 线切分为版面单元，黑线（`--ink`）划分大结构、灰线（`--line`）划分单元格，层级即线宽。全站唯一彩色是克莱因蓝 `#002FA7`，用于激活态、悬停反馈和编号——它出现的地方就是「系统正在响应你」的地方。零圆角、零阴影、零渐变（照片除外），一切软化手段都被禁用。中英混排是本风格的第二语言：英文超大加粗大写做视觉锤，中文小号大字距做信息层。

## 2. 色彩令牌

| Token | 浅色模式 | 深色模式 | 用途 |
|---|---|---|---|
| `--bg` | `#FFFFFF` | `#0C0C0C` | 页面背景 |
| `--ink` | `#111111` | `#F2F2F2` | 文字、主结构线、反色底 |
| `--sub` | `#767676` | `#8C8C8C` | 次要文字 |
| `--line` | `#E4E4E4` | `#262626` | 单元格分隔线 |
| `--blue` | `#002FA7` | `#3B62E0` | 克莱因蓝：激活/悬停/编号/CTA |
| `--blue-ink` | `#FFFFFF` | `#FFFFFF` | 蓝底上的文字 |

规则：除照片渐变外全站只允许这 5 个颜色变量出现；悬停反馈优先用「反色」（黑底白字）而非变色；蓝色系统性地只标记「可交互/正在交互」。深色模式将克莱因蓝提亮为 `#3B62E0` 以保证黑底可读。

## 3. 字体系统

| 角色 | 字体栈 | 使用位置 |
|---|---|---|
| 主字体 | `'Helvetica Neue', Helvetica, Arial, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif` | 全站 |
| 等宽 `.mono` | `'SF Mono', 'IBM Plex Mono', Consolas, monospace` | 编号、坐标、colophon |

字号阶梯：

| 层级 | 字号 | 行高 | 字重 | 其他 |
|---|---|---|---|---|
| H1 Hero | `clamp(44px, 7.2vw, 96px)` | 1.04 | 800 | 全大写，`letter-spacing: -.015em` |
| H1 中文副题 | `clamp(20px, 2.6vw, 34px)` | — | 400 | `letter-spacing: .3em` |
| 页脚大字 | `clamp(34px, 4.5vw, 56px)` | 1.1 | 800 | 全大写 |
| H2 栏目 | 22px | — | 800 | 大写；中文后缀 15px `.2em` |
| H3 条目 | 17px | — | 600–700 | — |
| 大数字 | 32–56px | 1 | 800 | 侧栏统计、作品序号 |
| 正文 | 15px | 1.7 | 400 | — |
| 标签 `.cap` | 10.5–12px | — | 400 | 全大写，`.12em–.18em` |

排版语法：英文标题负字距（紧），中文与标签正大字距（松）——一紧一松构成对比。

## 4. 栅格与线系统

| Token | 值 |
|---|---|
| 内容最大宽度 | 1240px（本风格最宽），两侧 32px |
| 主栅格 | 12 列 CSS Grid（Hero、照片区显式使用） |
| 结构黑线 | 1px `--ink`：header 底、每个 section 底、页脚分隔 |
| 单元灰线 | 1px `--line`：行分隔、列分隔、单元格 |
| 照片网格缝 | `gap: 1px; background: var(--line)`（缝即线） |
| 区块内 padding | 标题行 22px 上下；列表行 22–24px；单元格 28–44px |

页面不使用「区块间距」概念——相邻区块共享一条线，零外边距贴合。

## 5. 编号系统（签名元素）

全站内容统一编码，编号用等宽字体：

| 前缀 | 含义 | 示例 |
|---|---|---|
| `A1 / INDEX` | 页面坐标（Hero 眉头） | 配合经纬度 `31.23°N 121.47°E` |
| `01–04` | 栏目序号（sec-head 左列，蓝色） | `01 Writing` |
| `W–xxx` | 文章编号（倒序） | `W–042` |
| `01/02/03` | 作品序号（56px 大字水印） | — |
| `PL.xx` | 照片版号（图上白底标签） | `PL.01 — 外滩` |
| `K–XX` | 知识库分类码 | `K–FE` / `K–DS` / `K–RD` |

编号让内容像档案馆藏品一样可引用，也是四版中最强的「系统感」来源。落地时编号应由数据生成（如文章序号 = 总数递减），保持真实。

## 6. 组件规范

**导航**：sticky，高 58px，不透明背景 + 1px 黑色底线。品牌 `CHEN YU®`（® 蓝色上标）。链接全大写 12px：悬停黑底白字，激活蓝底白字（方角、无过渡圆滑感，`.15s`）。主题按钮 32px 方形 1px 黑描边，悬停反色。

**Hero**：12 列 grid。左 8 列：坐标标签行（1px 描边小方签，首个蓝色）→ 超大 H1（`Photo` 一词蓝色）→ 中文副题。右 4 列：三个统计单元格（上下排列，1px 灰线分隔），各含大数字 + 说明。底部横条：跑马灯式大写标语（静态）+ 蓝底实心 CTA `Start Reading`。

**文章索引（Writing）**：行式 grid `120px 1fr 200px 60px`（编号/标题/meta/箭头），1px 灰线分隔。悬停：整行浮 5% 蓝底、编号变蓝、箭头右移 6px。meta 为大写 `Design · 2026.07.14`。

**作品（Works）**：3 列等分，列间 1px 灰线（不留 gap）。单元格：56px 序号（默认 `--line` 色的"水印数字"）→ 大写标题 → cap 行（`Open Source · Rust · ★ 2.1k`）→ 描述。悬停：整格反色（黑底白字），序号变蓝——本风格最强的悬停语言。

**摄影（Plates）**：12 列 grid，`gap: 1px` 灰线缝。版式节奏：big（span 6，3:2）+ sm（span 3，3:4）混排。图片默认 `grayscale(35%)`，悬停恢复彩色 + `scale(1.04)`——「黑白档案被点亮」的隐喻。左上角白底版号标签 `PL.01 — 外滩`。

**知识库（Wiki）**：表格式行 `120px 1fr 1fr 140px 60px`：分类码（蓝）/ 大写标题 / 描述 / 笔记数（右对齐大写）/ 箭头。悬停同文章行。

**页脚**：两格 grid（1px 灰线分隔）：左格超大标语 `Let's build something honest.`（最后一词蓝色）；右格两列大写链接。底部 colophon 行：版权 + `Grid 12 col / Klein Blue 002FA7`（把规范本身写进页面）。

## 7. 动效

| 场景 | 参数 |
|---|---|
| 悬停反色/变色 | `.15–.2s`（本风格最快，干脆） |
| 行悬停底色 | `.15s` |
| 箭头位移 | `.2s`，右移 6px |
| 照片 | `.5s cubic-bezier(.2,.6,.2,1)`，scale + 去灰度 |
| 主题切换 | `.3s` |

无入场动画、无弹性曲线、无旋转——瑞士风的动效是开关，不是表演。

## 8. 响应式

| 断点 | 变化 |
|---|---|
| ≤900px | Hero 变单列，统计格横排；作品 1 列（右线改底线）；照片全部 span 6；文章/知识库行隐藏 meta 列；页脚单列 |
| ≤560px | 照片 span 12（单列） |

## 9. 无障碍

焦点：`outline: 2px solid var(--blue); offset 2px`。克莱因蓝 `#002FA7` 白底对比度 ≈ 10.7:1，可用于任意字号；深色模式 `#3B62E0` 黑底 ≈ 5.5:1，避免用于 12px 以下小字（编号 13px 达标）。反色悬停天然高对比。照片灰度处理不影响信息传达（版号标签始终清晰）。

## 10. Vue 落地提示

`IndexRow.vue`（通用行组件，props: cols 配置，Writing 与 Wiki 共用）、`WorkCell.vue`（反色悬停封装）、`PlateGrid.vue`（props: layout 数组控制 span 节奏）、`StatCell.vue`。编号生成器做成工具函数：`formatPostNo(i, total)` → `W–042`、`formatPlate(i)` → `PL.01`。线系统建议用 CSS Grid `gap:1px + background` 技法而非 border，避免双线叠加问题。
